# SQL--Retail-Insights-Data-Analyst
The Retail Insights Project is a SQL-based data analysis system designed for a supermarket chain (FreshMart).
It helps identify inventory issues such as overstocking, expiring products, and unsold items.
This project demonstrates how SQL can be used to generate data-driven inventory insights.

# Objectives
Analyze product stock and sales data

Identify products nearing expiration

Detect unsold ("dead stock") items

Calculate revenue contribution by category

# Database Schema
Tables Used:

Categories (category_id, category_name)

Products (product_id, product_name, category_id, price, stock_count, expiry_date)

SalesTransactions (transaction_id, product_id, quantity, sale_date)

# Tools & Technologies

SQL (SQLite)

Visual Studio Code

SQLite Extension

# Key SQL Analysis

1️⃣ Expiring Soon Products
Finds products that will expire within the next 7 days
Filters items with high stock (stock_count > 50)
Helps reduce wastage

2️⃣ Dead Stock Analysis
Identifies products with no sales in the last 60 days
Helps detect slow-moving or unsellable inventory

3️⃣ Revenue by Category
Calculates total revenue generated by each category
Uses SUM and GROUP BY
Helps identify top-performing categories
